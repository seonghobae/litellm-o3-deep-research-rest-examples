# PR continuity

This repository is small and typically prefers one focused PR per logical change.

## Canonical PR policy
- Reuse an existing open PR for the same branch when one already exists.
- Avoid duplicate PRs with the same scope.
- Keep documentation and code for the same example change together unless the docs are independently valuable.

## Merge readiness
- Python tests pass.
- Java tests pass.
- Any live verification performed for the change is captured in the PR description or commit history.
